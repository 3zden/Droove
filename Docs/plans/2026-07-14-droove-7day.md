## ¬ßContracts (single source of truth ‚Äî every task conforms to this section)

### Trip state machine (trip-service owns it)

```
REQUESTED ‚îÄ‚îÄ‚ñ∫ MATCHED ‚îÄ‚îÄ‚ñ∫ DRIVER_ARRIVED ‚îÄ‚îÄ‚ñ∫ IN_PROGRESS ‚îÄ‚îÄ‚ñ∫ COMPLETED
    ‚îÇ             ‚îÇ              ‚îÇ
    ‚ñº             ‚ñº              ‚ñº
 CANCELLED    CANCELLED      CANCELLED
    ‚îÇ
    ‚ñº (matching exhausts 3 rounds)
 NO_DRIVERS_FOUND
```

Legal transitions map (enforced in one place, `TripStateMachine.java`):

| From | To |
|---|---|
| REQUESTED | MATCHED, CANCELLED, NO_DRIVERS_FOUND |
| MATCHED | DRIVER_ARRIVED, CANCELLED |
| DRIVER_ARRIVED | IN_PROGRESS, CANCELLED |
| IN_PROGRESS | COMPLETED |

### Kafka topics & event envelope

| Topic | Partitions | Key | Producers ‚Üí Consumers |
|---|---|---|---|
| `trip-events` | 6 | tripId | trip-service ‚Üí notification-service, payment-service, analytics-sink |
| `match-requests` | 3 | tripId | trip-service, scheduling-service ‚Üí matching-service |
| `notifications` | 6 | targetUserId | matching-service ‚Üí notification-service |

Envelope on `trip-events` (verbatim schema, Java record `TripEvent` and Pydantic model `TripEvent` must match):

```json
{
  "v": 1,
  "eventId": "7f9c...uuid",
  "eventType": "TRIP_REQUESTED|TRIP_MATCHED|DRIVER_ARRIVED|TRIP_STARTED|TRIP_COMPLETED|TRIP_CANCELLED|NO_DRIVERS_FOUND",
  "tripId": "uuid",
  "riderId": "uuid",
  "driverId": "uuid-or-null",
  "occurredAt": "2026-07-15T18:04:05Z",
  "payload": { "fareCents": 2350, "surge": 1.2 }
}
```

`match-requests` message: `{"v":1,"tripId":"uuid","riderId":"uuid","pickupLat":33.5731,"pickupLng":-7.5898,"fareCents":2350}`
`notifications` message: `{"v":1,"targetUserId":"uuid","type":"DRIVER_OFFER|TRIP_UPDATE","payload":{...},"occurredAt":"..."}`

Payment consumption mapping (payment-service consumer group `payment-service`):
- `TRIP_MATCHED` ‚Üí `ESCROW_HOLD` (rider wallet ‚Üí escrow, `fareCents`)
- `TRIP_COMPLETED` ‚Üí `DISBURSE` (escrow ‚Üí driver 80%, escrow ‚Üí platform 20%; odd cent goes to platform)
- `TRIP_CANCELLED` ‚Üí `REFUND` (escrow ‚Üí rider wallet, only if a hold exists for the trip)
- Idempotency key for every ledger transaction = `eventId`.

### WebSocket protocols (FastAPI side)

- `WS /ws/location?token=JWT` ‚Äî role DRIVER. Client ‚Üí server every ~2s: `{"lat":33.57,"lng":-7.59,"heading":90,"ts":"..."}`. Server behavior = M1.
- `WS /ws/track/{driverId}?token=JWT` ‚Äî role RIDER. Server ‚Üí client: relayed position JSON from `driver:pos:{driverId}` pub/sub.
- `WS /ws/notifications?token=JWT` ‚Äî any role. Server ‚Üí client: `{"type":"DRIVER_OFFER"|"TRIP_UPDATE","payload":{...},"ts":"..."}` routed by userId from the two Kafka topics.

### Matching algorithm (matching-service; geo query is M1, orchestration is CLAUDE scaffold + M6)

On `match-requests` message: up to 3 rounds ‚Üí `findNearest(pickup, radiusKm=5, limit=5)` excluding already-offered drivers ‚Üí take nearest AVAILABLE ‚Üí CAS status AVAILABLE‚ÜíBUSY (Lua, provided) ‚Üí write `offer:{id}` hash + `offers:expiry` ZSET (+15s) ‚Üí publish `DRIVER_OFFER` notification. Accept ‚Üí `PATCH /trips/{id}/assign` ‚Üí done. Decline/timeout (expiry sweeper, 1s poll) ‚Üí CAS driver back to AVAILABLE ‚Üí next candidate. Rounds exhausted ‚Üí call trip-service internal `POST /trips/{id}/no-drivers` (‚ÜíNO_DRIVERS_FOUND).

CAS Lua (provided in scaffold, `matching-service/src/main/resources/cas.lua`):

```lua
if redis.call('GET', KEYS[1]) == ARGV[1] then
  redis.call('SET', KEYS[1], ARGV[2], 'KEEPTTL')
  return 1
end
return 0
```

### Ledger model (payment-service; M4)

```
accounts(id UUID PK, type RIDER_WALLET|DRIVER_WALLET|ESCROW|PLATFORM_REVENUE|EXTERNAL,
         owner_id UUID NULL, trip_id UUID NULL,        -- ESCROW rows: one per trip
         cached_balance_cents BIGINT NOT NULL DEFAULT 0)
ledger_transactions(id UUID PK, type ESCROW_HOLD|DISBURSE|REFUND|TOPUP,
         idempotency_key VARCHAR UNIQUE NOT NULL, trip_id UUID NULL, created_at)
ledger_entries(id UUID PK, tx_id FK, account_id FK, amount_cents BIGINT NOT NULL, created_at)
```

Invariants (each is a RED test in T21): (1) per-transaction `SUM(amount_cents) = 0`; (2) entries are append-only (`@Immutable`, no update/delete paths; prod note: `REVOKE UPDATE` ‚Äî documented, not built); (3) replaying the same `idempotencyKey` returns the original tx, writes nothing; (4) concurrent posts against one account serialize via `SELECT ‚Ä¶ FOR UPDATE` with accounts locked in sorted-UUID order (deadlock avoidance); (5) wallet/escrow accounts may not go negative (EXTERNAL may ‚Äî it's the money-in source).

### Fare formula (pricing-service, CLAUDE-owned)

`fare = max(700, round((500 + 120*distanceKm + 30*durationMin) * surge))` cents; `durationMin = distanceKm/30*60` (assumed 30 km/h); `distanceKm` = haversine; `surge = clamp(1.0 + 0.2*floor(cellCount/5), 1.0, 3.0)` where `cellCount = GET surge:{geohash5(pickup)} or 0`.

 *  sorted-UUID order; non-EXTERNAL accounts may not go negative; replay of
